Javascript หรือ js นั้นก็มีรูปแบบของ oop เพื่อใช้ในการพัฒนาเว็บไซต์เช่นดียวกับภาษาอื่นๆ prototype จะมีรูปแบบ ในรูปของ json objects โดยมี type เป็น json

Objects and Variables
โดยรูปแบบของ ตัวแปร หรือค่า variables ใน javascript นั้นจะมี type อยู่ไม่มากนักในกรณีนี้จะยกตัวอย่างของ type var

var name = 'yourname';

var age = 1;

จากตัวอย่างข้างต้นจะเห็นว่าเราได้ ตัวแปร type var สองตัวแปรคือ [name และ age] type var ไม่ใข่ strong type[strong type คือ type ที่ ระบุ type ตายตัวไม่สามารถรับ value อื่นได้นอกจาก type ที่เราได้ประกาศไว้เช่น int, string เป็นต้น] ดังนั้น var เราสามารถ set value อะไรก็ได้ เข้าไปในตัวแปรซึ่งในที่นี้ name จะมีtype เป็น string และ age มี type เป็น int ดังนั้น เราจะสามารถเปลี่ยนแปลงค่า variable เป็น value อะไรก็ได้เมื่อไหร่ก็ได้

var a = 'a';

console.log(a);

a = 1;

console.log(a);

จากตัวอย่างข้างต้นจะเห็นว่าเราจะได้ผลลัพธ์ log บรรทัดเเรกเป็น 'a' และบรรทัดที่สองจะได้เป็น 1 ซึ่งนี้คือการประกาศค่า variables ใน javascript

var obj = {name: 'yourname', age: 1};

จากตัวอย่างข้างต้นได้มีการ ประกาศตัวแปรที่มีชื่อว่า obj โดยภายในตัวแปรนั้นประกอบไปด้วย key name, age [เป็นในรูปแบบของ json value] ซึ่งนี่เป็นรุปแบบ objects แบบหนึ่งใน javascript โดย obj จะมีคุณสมบัติหลังจากการ set ค่าตัวแปรแล้วโดยมี property เป็น name และ age การเรียกใช้งานค่า key property ภายในตัวแปร obj

console.log(obj.name);

console.log(obj.age);

จากตัวอย่างข้างต้นจะเห็นว่าเราจะได้ผลลัพธ์ log บรรทัดเเรกเป็น 'yourname' และบรรทัดที่สองจะได้เป็น 1 และเราสามารถ เข้าถึง property โดยเขียนในรูปแบบ get ค่าแบบ array ได้เช่นกัน

console.log(obj["name"]);

console.log(obj["age"]);

Functions

function doSomethings() {

... do somethings

}

โดยสามารถเรียกใช้งาน functions ได้โดยการ

doSomethings();

Functions เป็นหนึ่งในส่วนประกอบหลักของ Javascript หนึ่งฟังก์ชันคือหนึ่งกระบวนการของ Javascript ซึ่งก็คือชุดคำสั่งที่ทำงานหรือคำนวณค่า โดยคุณต้องประกาศใช้ Functions ที่ใดก็ได้ภายในขอบเขตที่คุณเรียกใช้ โดยประกอบไปด้วย function ชือ่คำสั่ง ตามด้วย ชื่อของ functions ในที่นี้ก็คือ "dosomethings" และภายใน { } คือกระบวนการขั้นตอนการทำานของ functions วิธีการสร้าง functions มี สองแบบ 1 การสร้าง functions ให้ทำงานภายในขอบเขตนั้นๆ เช่นดั่งตัวอย่าง code ที่อยู่ในระดับ หรือ layer เดียวกัน จะสามารถ เรียกใช้ งานฟังก์ชั่นนี้ได้ ซึ่งขอบเขตในที่นี้เรียกว่า Scope ซึ่งจะพูดใน หัวข้อต่อไป อีกวิธีการสร้าง functions คือการ สร้าง functions ภายใน objects หรือ สร้างเป็น variables

var doSomethings = function() {

... do somethings

}

จาก fucntions นี้ เราจะได้ variables doSomethings ที่เป็น ประเภท functions โดยสามารถเรียกใช้งาน functions ได้โดยการ

doSomethings();

var obj = {name: 'yourname', age: 1};

obj.doSomethings = function() {

... do somethings

}

เราจะได้ fucntions doSomethings ที่เป็น ประเภท functions ใน objects obj โดยสามารถเรียกใช้งาน functions ได้โดยการ

obj.doSomethings();

var obj = {name: 'yourname', age: 1};

obj.doSomethings = function() {

... do somethings

}

เราจะได้ fucntions doSomethings ที่เป็น ประเภท functions ใน objects obj โดยสามารถเรียกใช้งาน functions ได้โดยการ

obj.doSomethings();

Scope

var me = {name: 'myname'};

var you = {name: 'yourname'};

me.whatName = function() {

return "Hi, I'm " + this.name;

}

you.whatName = me.whatName;

การเขียน Scope จะเจอบ่อยๆในการเขียน Javascript. Scope เป็นการ จำกัดขอบเขตของการเรียกใช้งาน variable functions หรือ objects เพื่อให้เกิดการทำงานที่มีประสิทธิภาพในกระบวนการทำงานของ Program จากตัวอย่างข้างต้น จะเห็นว่าเรามี objects 2 ตัว คือ me และ you ซึ่งในแต่ละ objects ประกอบด้วย property คือ name และ ได้มีการสร้าง functions ให้กับ objects me คือ whatName โดยการทำงานภายใน Functions นั้น คือการ ส่งค่าออกจาก Functions เป็นค่า string "Hi, I'm " + this.name; และได้มีการสร้างโยน ค่าจาก objects me โดยโยนค่า functions whatName ให้กับ objects you เป็นเหมือนการ clone function ให้กันจุดสำคัญของ โปรแกรมชุดนี้คือ return "Hi, I'm " + this.name; this ในที่นี้หมายถึง ตัวของ objects ใน function นั้นๆ ก็คือ function ของ me ก็จะได้ objects เป็น me ซึ่งมีการเเรียก property name ดังนั้น ค่าที่จะได้จากการเรียก fucntions whatName ของ objects me ก็คือ 'myname'

console.log(me.whatName()); // result : 'mename'

console.log(you.whatName()); // result : 'yourname'

นอกจากนี้เรายังสามารถใช้หลักการการทำ scope เพื่อใช้ในการป้องกันหรือทำ encapsulation ให้กับโปรแกรมของเราได้โดยทุกอย่างที่อยู่ภายใน scope จะไม่สามารถ access จาก ข้างนอกได้ เพื่อป้องกันไม่ให้เกิดความเสียหายกับ value ที่จะใช้ในการะบวนการทำงานภายใน scope นั้นๆ

(function funcC() {

var a = 1;

console.log(a);

})();

// result: 1 [immediately-invoked function expression (IIFE)]

funcC();

// Uncaught ReferenceError: funcC is not defined

console.log(a);

// Uncaught ReferenceError: a is not defined
